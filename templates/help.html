Sample ID Handling

Sample IDs are optional. The application handles missing sample IDs intelligently:

With Sample IDs Provided
Quadrants are labeled with the sample IDs you enter (comma-separated). Example: Sample1, Sample2, Sample3
Without Sample IDs (Left Blank)
Linear interpolation (NT50/NT90): Quadrants are left blank in the Excel file. Appear as Unlabelled on the Summary Sheet.
Curve fitting (IC50): Samples are automatically labeled as Unlabelled1, Unlabelled2, etc. This ensures all quadrants are processed and identifiable.

Analysis Methods
Linear Interpolation (Default Method)
Overview: Calculates NT50 and NT90 titres by finding the dilution at which the luminescence signal crosses 50% or 90% of the no-serum control (NSC) value.
Procedure: For each of the three replicate columns within a quadrant, the target luminescence (50% or 90% of the NSC value for that replicate) is identified. The two dilution points flanking this target are located, and the corresponding dilution is estimated by linear interpolation between them. The three per-replicate titres are then averaged to produce a single NT50 or NT90 value per quadrant.
Boundary handling: If all measured luminescence values lie below the target threshold (strong neutralisation across the entire range), the titre is reported as ≤ the lowest dilution tested. If all values exceed the threshold (no detectable neutralisation), the titre is reported as ≥ the highest dilution tested.
Output: NT50 and NT90 titres for each quadrant, reported as triplicate values and averages in the Excel "Data Summary" sheet.
Sigmoid Curve Fitting
Overview: Fits a four-parameter logistic (4PL) sigmoid function to percentage neutralisation data, yielding an IC50 value and associated quality metrics for each sample–virus combination.
Normalisation: Raw luminescence values are first converted to percentage neutralisation using the formula: % Neutralisation = 100 × (1 − x̄dilution / x̄NSC), where x̄dilution is the triplicate mean at a given dilution and x̄NSC is the no-serum control mean. Dilution factors are transformed to −log₂ scale.
Model fitting: The sigmoid f(x) = Lower + (Upper − Lower) / (1 + exp(−Slope × (x − IC50))) is fitted using the Levenberg–Marquardt algorithm (minpack.lm R package), with bounded parameter constraints to ensure biologically plausible solutions. Goodness of fit is assessed by pseudo-R². Samples with R² < 0.5 or failed convergence are classified as flat titrations and represented by a horizontal line at the mean neutralisation.
Output: IC50 values (log₂ dilution at 50% neutralisation), reciprocal dilution titres (2−IC50), R² statistics, quality flags, and fitted curve plots for each virus. Full methodological detail is provided on the Curve Fitting Results page.
Titre Comparison Analysis
Overview: Compares NT50 titres from linear interpolation with IC50-derived titres from sigmoid curve fitting to assess agreement between the two methods.
Procedure: Titre values from both methods are matched by pseudotype and sample identifier. Both sets of titres are log₁₀-transformed, and Pearson's correlation coefficient (r) and R² are computed via linear regression. Percentage disagreement is calculated for each matched pair as |NT50 − IC50 Titre| / NT50 × 100.
Output: Scatter plot of NT50 vs IC50 titres (with identity line and regression fit), summary statistics (correlation, R², disagreement rate), and a table of the top 5 mismatches ranked by percentage disagreement. Full methodological detail is provided on the Titre Comparison Results page.

Technical Details

For those interested in the technical implementation:

Neutralisation Calculation (for sigmoid fitting)
% Neutralisation = 100 × (1 − x̄dilution / x̄NSC)
Sigmoid Function (4-Parameter Logistic)
f(x) = Lower + (Upper − Lower) / (1 + exp(−Slope × (x − IC50)))
Titre Derivation from IC50
Titre = 2−IC50
Example: IC50 = −8 (log₂) → Titre = 2⁸ = 256
Dilution to Log₂ Conversion
DilutionLog2 = −log₂(Dilution Factor)
Example: Dilution factor of 1/256 → −log₂(1/256) = 8
Goodness of Fit (Pseudo-R²)
R² = max(0, 1 − RSS / TSS)
RSS = residual sum of squares; TSS = total sum of squares about the mean
Software Stack
Backend: Python (Flask), openpyxl
Curve fitting: R (tidyverse, minpack.lm — Levenberg–Marquardt algorithm)
Visualisation: R (ggplot2, cowplot, plotly)
Interface: HTML5, Bootstrap 5, JavaScript
